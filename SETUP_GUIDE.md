# Setup Guide — Your GitHub Profile README

This folder is ready to push as-is. It already has your real info and real links filled in — no placeholders left.

## 1. Create the special profile repository

GitHub shows a README as your profile page only if the repo is named **exactly** your username:

```
usamagulzar/usamagulzar
```

Go to https://github.com/new and create a public repository named `usamagulzar` (no description needed, don't initialize with a README — you already have one here).

## 2. Push these files

From inside this folder:

```bash
git init
git add .
git commit -m "Initial profile setup"
git branch -M main
git remote add origin https://github.com/usamagulzar/usamagulzar.git
git push -u origin main
```

## 3. Turn on the contribution graph animation

The falling-tetromino animation (near the bottom of the README) is generated by a GitHub Action already included in `.github/workflows/snake.yml` (the filename is unchanged, but the workflow itself now generates the tetromino graph instead of a snake). To let it run and commit the SVG back to your repo:

1. Go to your new repo → **Settings → Actions → General**
2. Under "Workflow permissions," select **Read and write permissions**
3. Save, then go to the **Actions** tab and manually run "Generate Contribution Graph Art" once (it also runs automatically every 24 hours and on every push)

It writes to a branch called `output` automatically — you don't need to create that branch yourself.

## 4. Double check

Visit `https://github.com/usamagulzar` — your new profile README should be live immediately after the push (the banner and contribution graph may take a minute to render the first time, and the contribution graph won't appear at all until the Action has run once).

## A note on the banner images

The banner uses `raw.githubusercontent.com/usamagulzar/usamagulzar/main/assets/...` URLs, which only resolve once you've actually pushed this `assets/` folder to your `main` branch on GitHub — they won't preview locally before that. If you ever rename the default branch (e.g. to `master`), update `main` in those two URLs in `README.md` to match.

If you want to tweak the banner's wording later (e.g. swap "Open to internships" once you land one), just edit the `<text>` content in `assets/banner-dark.svg` and `assets/banner-light.svg` — both files are plain, readable SVG markup.
