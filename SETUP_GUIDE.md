# Setup Guide — Your GitHub Profile README

This folder is ready to push as-is. It already has your real info and real links filled in — no placeholders left.

## 1. Create the special profile repository

GitHub shows a README as your profile page only if the repo is named **exactly** your username:

```
usamagulzar/usamagulzar
```

Go to https://github.com/new and create a public repository named `usamagulzar` (no description needed, don't initialize with a README — you already have one here).

## 2. Push these files

From inside this folder:

```bash
git init
git add .
git commit -m "Initial profile setup"
git branch -M main
git remote add origin https://github.com/usamagulzar/usamagulzar.git
git push -u origin main
```

## 3. Turn on the contribution graph animation

The falling-tetromino animation near the bottom of the README is generated by a GitHub Action already included in `.github/workflows/snake.yml` (the filename is unchanged, but the workflow itself generates the tetromino graph, not a snake). To let it run and commit the SVG back to your repo:

1. Go to your new repo → **Settings → Actions → General**
2. Under "Workflow permissions," select **Read and write permissions**
3. Save, then go to the **Actions** tab and manually run "Generate Contribution Graph Art" once (it also runs automatically every 24 hours and on every push)

It writes to a branch called `output` automatically — you don't need to create that branch yourself.

## 4. Double check

Visit `https://github.com/usamagulzar` — your profile README should be live immediately after the push. The contribution graph won't appear until the Action has run at least once (step 3).

## What's in this version, and why

- **Badges use one unified color** (`#24292F`, a dark slate) across every category instead of each tech's official brand color. The rainbow-of-brand-colors look (Python blue next to Java red-orange next to JS yellow) is what made the badge rows read as templated. One consistent palette makes it read as a designed system instead.
- **Highlights are a badge row, not a table** — tables for 3–4 short stats tend to render cramped on GitHub; a badge strip in the same palette as the Stack section keeps it visually consistent and easy to scan.
- **No custom banner graphic** — name + role + contact badges in the same palette does the job without an SVG asset to maintain.
- Pinned repos (shown automatically by GitHub below this README) act as your project showcase, so there's no hardcoded project list to keep updating.

If you'd like the accent color changed from slate to something else, every badge uses the same hex (`24292F`) — swap that one value throughout `README.md` and the whole page re-themes at once.
